# 新旧仓库评测对比（一页摘要）

## 核心结论

- 新仓库相对旧仓库在**复杂检索场景**（高干扰、大样本）有**明确且显著**提升。
- 在简单场景（`s8,d10`）表现为**持平**，没有回退，也不应夸大为“全场景提升”。
- 因此可确认：**当前改造后的新仓库整体检索能力优于旧仓库**。

### 提升幅度（看 HR@10）

- `s8,d200`：
  - C：`0.313 -> 0.563`（`+0.250`，约 `+80.0%`）
  - D：`0.375 -> 0.625`（`+0.250`，约 `+66.7%`）
- `s100,d200`：
  - C：`0.280 -> 0.580`（`+0.300`，约 `+107.1%`）
  - D：`0.295 -> 0.615`（`+0.320`，约 `+108.5%`）

## 时延与稳定性（简版）

- 质量提升主要来自检索有效性，不是单纯靠放宽门禁。
- `phase6` 门禁在本轮对比中均通过（`valid=true`）。
- 新仓增强配置 `cm=8`（`s100,d200` 连跑 3 次）可进一步提升质量到：
  - C：`HR@10≈0.700`
  - D：`HR@10≈0.720`
- 但 `cm=8` 会显著提高 D 侧延迟（质量-时延权衡）。

---

## 管理决策建议

1. 若业务优先“命中率/召回质量”：高干扰场景优先使用新仓增强配置（如 `cm=8`）。
2. 若业务优先“时延”：默认保留 `cm=4`，在高干扰流量上按策略切换到 `cm=8`。
3. 评测制度建议固定：每次版本发布至少保留 `s8,d10`、`s8,d200`、`s100,d200` 三组同口径对比。
