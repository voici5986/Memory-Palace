## 🌐 通用操作

### 🧭 顶部导航栏

页面顶部有四个选项卡，点击即可切换页面：

| 选项卡 | 对应页面 | 一句话说明 |
|--------|---------|-----------|
| **记忆** | 记忆页面 | 浏览、创建、编辑记忆 |
| **审查** | 审查页面 | 查看每次修改的变更记录，决定保留还是回滚 |
| **维护** | 维护页面 | 清理不再需要的旧记忆 |
| **观测** | 观测页面 | 查看系统运行状态和搜索性能 |

### 🔑 设置 API 密钥

页面右上角通常会出现下面三种状态之一：

- **设置 API 密钥**：浏览器里还没有存本地 key；点击后会打开**首启配置向导**
- **更新 API 密钥 / 清除密钥**：浏览器里已经存了一把本地 key
- **运行时密钥已启用**：页面拿到了运行时注入的 key；这种情况下通常不需要手动再输，但右上角仍可打开 **配置向导** 查看当前状态或填写本地 `.env` 所需字段

如果浏览器里既没有已保存的 key，也没有运行时注入的 key，首启配置向导可能会在第一次打开页面时自动弹出。

如果 Dashboard 外壳已经打开，但受保护的数据还是没加载出来，最常见的处理方式是：

点击 **设置 API 密钥** → 打开首启配置向导 → 在里面输入你在 `.env` 文件中配置的 `MCP_API_KEY` 值 → 选择 **只保存 Dashboard 密钥** 或 **保存到本地 `.env`**。

> 如果你在本地调试时设置了 `MCP_API_KEY_ALLOW_INSECURE_LOCAL=true`，并且请求是直连 loopback 地址，页面也可以不手动输入密钥就直接显示受保护数据。

> **只保存 Dashboard 密钥** 会把这把 key 保存在当前浏览器会话里（`sessionStorage`），直到你手动清除或这次浏览器会话结束。向导里的 `Profile C/D` 预设现在按文档口径走 `router + reranker` 路线；如果你本机的 router 还没准备好，就手动把检索字段切回直连 API 模式。

> 如果你选择的是 **保存到本地 `.env`**，并且同时填了 Dashboard key，要记住 `.env` 写入和浏览器 key 保存是两步。现在只要浏览器本地存储失败，向导就会明确提示保存失败，不再给出误导性的成功提示。实际使用时，这通常意味着 `.env` 可能已经写进去了，但浏览器侧鉴权还没准备好。

> 向导里的 `.env` 写入能力只在“本地 checkout + 非 Docker 运行”时开放；如果当前页面连的是 Docker 容器，它会明确停留在说明模式，不会假装已经持久化容器 env / 代理配置。

> 如果你 fresh first-run 打开时先看到英文弹窗，也不用慌。向导右上角自己就有语言切换按钮，可以直接切到中文。

## 📂 记忆页面

> **这个页面是干什么的？** 浏览和管理你所有的记忆内容。你可以在这里查看已有的记忆、创建新记忆、编辑内容、或者删除不再需要的记忆。

### 🗂️ 页面布局

页面分为两大区域：

- **左侧 — 对话记忆仓**：用来创建新的记忆
- **右侧 — 当前节点内容 + 子记忆列表**：查看和编辑已有的记忆

### 🧭 面包屑导航（页面顶部路径条）

页面最上方是面包屑导航条，显示你当前所处的位置，例如：`根 > core > agent > preferences`。

- 点击路径中任意一级可以直接跳转到对应层级
- **根** 是最顶层节点，根节点本身不存储记忆内容

### ✍️ 左侧：对话记忆仓

这是创建新记忆的入口。

#### 📝 各字段说明

| 字段名 | 界面显示 | 说明 |
|--------|---------|------|
| **记忆标题** | `记忆标题（可选）` | 给这条记忆起一个简短的名字，方便之后查找。可以不填，但建议填写。 |
| **对话内容** | 大文本框（灰色示例文字） | 把你想保存的 LLM / Agent 对话内容粘贴到这里。这是必填项。 |
| **优先级** | `优先级` | 填一个数字。**数字越小，优先级越高**；`0` 比 `5` 更靠前。留空则使用默认值。 |
| **披露条件** | `披露条件` | 控制这条记忆在什么场景下可以被 Agent 自动获取到。例如填 `"仅在用户明确询问时展示"` 或留空表示不限制。这是一个可选的文本描述字段。 |

> **什么是"披露条件"？** 简单来说：你可以在这里写一段文字，告诉系统"这条记忆应该在什么条件下被展示给 AI"。比如你存了一条敏感偏好，你可以注明"只有用户主动问起来才展示"。如果不确定该填什么，留空即可。

#### ✅ 操作步骤

1. 在 **对话内容** 文本框中粘贴你想保存的对话
2. （可选）填写 **记忆标题**、**优先级**、**披露条件**
3. 点击 **写入记忆** 按钮
4. 系统会先经过 Write Guard（写入守卫）检查，判断是否应该创建这条记忆
5. 如果创建成功，页面底部会显示绿色提示 `记忆已创建：core://xxx`

> **什么是 Write Guard（写入守卫）？** 这是系统在接受写入前做的一层安全检查。站在 Dashboard 页面这一侧，你能稳定看到的是“这次写入是否继续执行”，或者返回 `已跳过：write_guard 阻止了 ...`。更细的原因仍以后端返回结果为准。

### 📄 右侧上方：当前节点内容

这里显示你当前正在查看的那条记忆的详细内容。

- **内容编辑区**：可以直接修改记忆的正文内容
- **优先级**：可以修改这条记忆的优先级数字
- **披露条件**：可以修改披露条件
- **保存** 按钮：修改后点这个保存变更
- **删除路径** 按钮：删除这条记忆的访问路径

> 点击 **删除路径** 时，系统会先弹出确认框，告诉你即将删除的路径。确认后才会执行删除。

#### 📌 Gist（摘要 / 原文视图）

每条记忆可能有一个系统自动生成的摘要（Gist）。你可以通过 **摘要** / **原文** 切换按钮在摘要视图和完整原文之间切换查看。

- **摘要**：系统自动压缩生成的简要版本，方便快速了解内容
- **原文**：记忆的完整原始内容

### 📑 右侧下方：子记忆列表

显示当前节点下的所有子记忆卡片。

#### 🔍 子节点筛选

在子记忆列表上方有筛选工具：

| 筛选项 | 界面显示 | 说明 |
|--------|---------|------|
| **搜索框** | `搜索路径 / 摘要` | 输入关键字，按路径和摘要片段快速过滤出匹配的子记忆 |
| **最高优先级** | `最高优先级（可选）` | 填一个数字，只显示优先级小于等于该值的记忆 |

- 每张子记忆卡片会显示当前优先级、路径/标题，以及一段摘要或正文片段
- 点击任意子记忆卡片可以进入该节点查看详情
- 列表会先把当前节点已拿到的子项放到前端，再通过底部的 **再加载 N 条** 按钮按批次继续展开

> 如果你正在编辑当前节点的内容，这时再去点面包屑或子记忆卡片，页面会先提醒你是否丢弃未保存编辑。

## 📋 审查页面

> **这个页面是干什么的？** 每当记忆被创建、修改或删除时，系统都会记录一份"修改前的快照"。这个页面让你看到这些变更，你可以选择"确认这个变更"或"回滚到修改前的状态"。
>
> **作用域提醒**：审查页默认只显示**当前数据库**对应的快照会话。如果你切换了本地 `.env`、换了 Docker compose project，或者改到了另一份 SQLite 文件，旧库里的会话不会继续混进当前列表。

### 🗂️ 页面布局

- **左侧 — 审查账本**：历史会话列表和每个会话中的快照
- **右侧 — 差异对比视图**：选中某条快照后，展示修改前后的详细对比

### 📖 左侧：审查账本

#### 🎯 目标会话

顶部下拉框显示 **目标会话**。每次 AI 操作产生的一组修改会归入同一个会话。

- 如果显示 **暂无活跃会话**，说明目前没有待审查的变更
- 如果你刚切换过数据库、Docker volume 或 profile，旧环境里的会话默认不会继续显示在这里
- 选择一个会话后，下方会列出该会话中所有的修改快照

#### 📸 快照列表

每条快照是一次具体的修改操作，卡片上会显示：
- **操作类型**：创建 / 内容修改 / 元数据修改 / 删除 / 别名
- **资源路径**：被修改的记忆路径（例如 `core://agent/preferences`）

点击某条快照，右侧就会显示这次修改的详细对比。

### 🔀 右侧：差异对比

选中快照后，这里展示修改前后的内容差异：

- **红色/删除线部分**：被删除的内容
- **绿色/新增部分**：新增加的内容
- **元数据变化**：如果优先级、披露条件等元数据发生了改变，也会在这里展示

#### 🎛️ 操作按钮

| 按钮 | 界面显示 | 作用 |
|------|---------|------|
| **整合** | `整合` | 确认这次修改没问题，接受变更并清除这条快照 |
| **拒绝** | `拒绝` | 不接受这次修改，把记忆回滚到修改前的状态 |
| **整合全部** | `整合全部` | 一次性确认当前会话中所有待审查的变更 |

> **"整合"是什么意思？** 可以理解为"确认通过"。点击后，系统认为你已经审核过这次变更，快照记录会被清理掉。如果你发现这次修改有问题，就用"拒绝"来回滚。

> **"拒绝/回滚"有什么后果？** 点击"拒绝"后，被修改的记忆会恢复到修改之前的状态。已经删除的内容会恢复回来，新增的内容会被撤销。

## 🔧 维护页面

> **这个页面是干什么的？** 帮你清理不再需要的"垃圾记忆"。随着时间推移，有些记忆会变得过时、不再被任何路径引用、或者活力值衰减到很低。这个页面帮你找出并安全地清理它们。

页面顶部显示副标题：*集中处理孤儿记忆与低活力清理候选，并保留人工确认步骤。*

### 📊 顶部统计卡片

页面上方有三个统计卡片：

| 卡片 | 含义 |
|------|------|
| **已弃用** | 记忆被更新后留下的旧版本数量（类似文件的历史版本） |
| **孤立记忆** | 没有任何路径可以访问到的记忆数量（"找不到入口"的记忆） |
| **低活力** | 活力值已经衰减到很低、可以被清理的记忆数量 |

### 🧹 上半区：孤儿记忆清理

> **什么是"孤儿记忆"？** 在 Memory Palace 中，每条记忆通过一个"路径"来访问（例如 `core://agent/preferences`）。如果这个路径被删除了，记忆本身可能还在数据库里，但没有人能访问到它——这就叫"孤儿记忆"。

这个区域列出所有孤儿记忆，分为两类：

- **已弃用版本**：旧版本记忆（更新操作产生的历史副本）
- **孤立记忆**：没有任何路径指向的记忆

每张卡片都可以展开查看完整内容；只有“已弃用版本”并且带有 `migration_target` 的条目，才会额外显示差异对比。

#### ✅ 操作步骤

1. 点击 **刷新** 扫描最新的孤儿记忆
2. 勾选你想删除的记忆（也可以使用分组标题旁边的全选控件）
3. 点击 **删除 N 条孤儿记忆**
4. 在弹出的确认框中确认删除

> ⚠️ 删除操作是永久性的，无法撤销。请在删除前仔细查看内容，确认确实不再需要。

### 💚 下半区：活力清理候选

> **什么是"活力值"？** 每条记忆都有一个"活力值"（vitality score），代表这条记忆有多"活跃"。新创建的记忆活力值最高，之后会随时间自然衰减。如果一条记忆很久没被访问，它的活力值就会降得很低，系统会把它标记为"可清理候选"。

#### 🔍 筛选条件

| 字段 | 界面显示 | 说明 |
|------|---------|------|
| **阈值** | `阈值` | 活力值的下限。系统会找出活力值低于这个数字的记忆。数字越小，筛选越严格。 |
| **非活跃天数** | `非活跃天数` | 多少天没被访问过的记忆才被列入候选。例如填 14 表示只列出 14 天以上没被访问的记忆。 |
| **数量上限** | `数量上限` | 最多显示多少条候选结果。范围 1~500。 |
| **域** | `域` | 只筛选某个特定域（例如 `core`、`notes`）下的记忆。留空表示不限。 |
| **路径前缀** | `路径前缀` | 只筛选路径以某个前缀开头的记忆。例如填 `agent/` 表示只看 `agent/` 下面的记忆。 |
| **审核人** | `审核人` | 记录是谁发起了这次清理操作。留空也可以；默认值是 `maintenance_dashboard`。 |

#### ✅ 操作步骤

1. 填写筛选条件，点击 **应用筛选**
2. 查看下方列出的候选记忆列表
3. 每条候选卡片上会显示：
   - `vitality N.NN`：当前活力值
   - `inactive Nd`：已经多少天没被访问了
   - `可删除` 或 `仍有活动路径`：是否可以安全删除
4. 勾选你要处理的记忆
5. 选择操作：
   - **准备删除（N）**：将选中的记忆标记为删除
   - **准备保留（N）**：将选中的记忆标记为保留（不删除）
6. 点击 **确认 删除** 或 **确认 保留** 后，系统会要求你输入一段确认短语（安全机制，防止误操作）
7. 输入正确的确认短语后，操作才会真正执行

> **"执行衰减并刷新"按钮是做什么的？** 手动触发一次活力值衰减计算。正常情况下系统会自动做，但如果你想立即看到最新的衰减结果，可以点击这个按钮。

## 📊 观测页面

> **这个页面是干什么的？** 这是系统的"体检报告"页面。它显示搜索引擎的运行状况、性能数据、和后台任务的状态。**普通用户日常不需要频繁使用这个页面**，它更适合排查问题时使用。

页面顶部显示副标题：*跟踪检索延迟、降级原因、缓存命中与索引任务健康状态。*

### 📊 顶部统计卡片

| 卡片 | 含义 |
|------|------|
| **查询总数** | 系统总共处理了多少次搜索请求 |
| **延迟** | 搜索请求的平均响应时间（毫秒），越低越好 |
| **缓存命中率** | 搜索结果从缓存中获取的比例，越高越好（说明重复查询不需要重新计算） |
| **索引延迟** | 索引操作的平均耗时 |
| **清理 p95** | 清理操作中 95% 的请求在这个时间内完成（用来衡量最慢的情况有多慢） |
| **清理索引命中率** | 清理操作中索引的命中率 |

### 🎛️ 顶部操作按钮

| 按钮 | 界面显示 | 说明 |
|------|---------|------|
| **刷新** | `刷新` | 重新加载页面上的所有统计数据 |
| **重建索引** | `重建索引` | 向后端发起一次全量索引重建作业请求。搜索效果明显变差时可以尝试。 |
| **睡眠整合** | `睡眠整合` | 向后端发起一次碎片记忆整合作业请求。 |

> **什么是"索引重建"？** 这个按钮会向后端提交一次重建请求。如果你怀疑索引过期了、或者搜索结果明显不对，可以把它理解成“让后端重新生成目录”。
>
> **什么是"睡眠整合"？** 这个按钮会向后端提交一次整合作业请求。它的目标是帮助系统整理碎片化记忆，但具体效果仍以后端当前策略和运行时状态为准。

### 🔎 搜索控制台

这是一个诊断工具，让你亲自跑一次搜索来测试系统的检索效果。

#### 📝 各字段说明

| 字段 | 界面显示 | 说明 |
|------|---------|------|
| **查询词** | `查询词` | 你想搜索的关键字或问题 |
| **模式** | `模式` | 检索模式。可选：混合（推荐，综合关键词和语义）、语义（基于含义搜索）、关键词（精确匹配） |
| **会话 ID** | `会话 ID（可选）` | 填入则优先返回该会话相关的结果 |
| **最大结果数** | `最大结果数` | 最多返回多少条搜索结果 |
| **候选倍数** | `候选倍数` | 内部搜索时的候选放大倍数。数字越大，搜索范围越广但越慢。默认值通常够用。 |
| **优先会话结果** | `优先会话结果` | 勾选后，当前会话中的记忆会排在搜索结果前面 |
| **域过滤** | `域过滤（可选）` | 只在某个特定域中搜索，例如 `core` |
| **路径前缀过滤** | `路径前缀（可选）` | 只搜索某个路径前缀下的记忆 |
| **范围提示** | `范围提示（可选）` | 给搜索引擎的提示信息，帮助它更精准地定位搜索范围。例如 `core://agent` |
| **最大优先级过滤** | `最大优先级（可选）` | 只返回优先级不超过此值的记忆 |

点击 **执行诊断搜索** 后，下方会显示搜索结果和诊断信息。

### 🩺 搜索诊断

执行搜索后，这里会展示：

- **延迟**：本次搜索花了多少毫秒
- **模式**：实际使用的检索模式
- **意图**：后端返回的查询意图或实际采用的意图标签
- **策略**：系统选择了哪种搜索策略
- **降级**：本次搜索是否被后端标记为降级；如果发生降级，这里会显示具体原因
- **结果列表**：每条结果显示匹配分数、内容摘要、来源路径和更新时间相关信息

### ⚙️ 运行时快照

显示系统当前的运行状态，包括：
- 索引是否正常或降级
- 队列深度（还有多少任务在排队）
- 最近一次错误信息
- 睡眠整合状态

### 📋 索引任务队列

显示正在执行或最近完成的索引任务列表。每条任务可以：
- **查看**：查看任务详细信息
- **取消**：中止正在进行的任务
- **重试**：重新执行失败的任务

## ❓ 常见问题

### Q: 页面打开后什么数据都没有？

页面能打开但看不到数据，通常是因为 **还没配置 API 密钥**。点击右上角 **设置 API 密钥** 打开首启配置向导，输入你在 `.env` 中配置的 `MCP_API_KEY`，然后先用“只保存 Dashboard 密钥”让浏览器通过鉴权即可。只有在“本地 checkout + 非 Docker 运行”场景下，才建议顺手用它写 `.env`。如果“保存到本地 `.env`”提示失败，一个常见情况是后端 `.env` 已经写好了，但浏览器还是没把 Dashboard key 存下来。

### Q: 点击"写入记忆"后提示"已跳过"？

这说明 Write Guard（写入守卫）阻止了这次写入。站在 Dashboard 页面这一侧，不一定总能直接看到后端的具体理由；你可以先修改内容后重试，或者再去看后端返回结果。

### Q: "优先级"应该填多大的数字？

没有固定范围。建议用一个简单的规则：非常重要的核心记忆填 `0`，普通记忆填 `5`，更靠后的弱提示信息再用更大的数字。记住：**数字越小越重要**。

### Q: "整合"和"拒绝"操作能撤销吗？

- **整合** 后快照记录会被清除，但记忆内容本身不会变化
- **拒绝** 会把记忆回滚到修改前的状态；如果回滚后发现还是需要这个修改，需要重新创建

### Q: 活力值会自动恢复吗？

不会自动恢复。活力值只会随时间衰减。但每次记忆被访问（读取或搜索命中）时，活力值会得到一定程度的"刷新"，减缓衰减速度。

### Q: 什么时候需要手动"重建索引"？

当你发现搜索不出明明存在的记忆时，可以尝试重建索引。正常使用情况下，索引会自动维护。
